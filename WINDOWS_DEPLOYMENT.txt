FOROBS v3 — Windows Deployment Manual
=======================================

Goal: Single .exe file. No source code visible. No Python installation.
       No admin rights needed. No internet on ship PC.
       User double-clicks FOROBS.exe → app opens in browser.


1. WHAT THE USER SEES ON SHIP PC
==================================

    FOROBS-v3/
    ├── FOROBS.exe           ← double-click to run (everything inside)
    ├── logbook.db           ← database (auto-created on first run)
    ├── card_settings.json   ← card dimensions (editable if needed)
    ├── card_layout.json     ← card positions (auto-saved)
    └── chart_data.json      ← chart cache (auto-created)

No .py files, no Python folder, no source code — nothing to edit or break.


2. BUILD PROCESS (needs a Windows PC with internet — ONE TIME)
================================================================

2.1 Install Python 3.12 on the build PC
-----------------------------------------
Download from python.org, install normally. This is the BUILD machine only.

2.2 Create build environment
------------------------------
    mkdir C:\FOROBS-build
    cd C:\FOROBS-build
    python -m venv venv
    venv\Scripts\activate
    pip install streamlit pandas openpyxl plotly pyinstaller

2.3 Copy source files to build folder
---------------------------------------
Copy from USB/project:
    app.py
    transfer_agent.py
    auto_transfer.py
    launcher.py
    card_settings.json
    card_layout.json
    requirements.txt

2.4 Create the launcher wrapper
---------------------------------
Create file: forobs_launcher.py

    import subprocess, sys, os, time, webbrowser, threading

    def open_browser():
        time.sleep(3)
        webbrowser.open("http://localhost:8501")

    if __name__ == "__main__":
        os.chdir(os.path.dirname(os.path.abspath(sys.argv[0])))
        threading.Thread(target=open_browser, daemon=True).start()
        subprocess.run([
            sys.executable, "-m", "streamlit", "run", "app.py",
            "--server.port", "8501",
            "--server.headless", "true",
            "--browser.gatherUsageStats", "false",
            "--server.address", "0.0.0.0",
        ])

2.5 Build the .exe
--------------------
    pyinstaller --onefile --name FOROBS ^
        --add-data "app.py;." ^
        --add-data "transfer_agent.py;." ^
        --add-data "auto_transfer.py;." ^
        --add-data "card_settings.json;." ^
        --add-data "card_layout.json;." ^
        --hidden-import streamlit ^
        --hidden-import pandas ^
        --hidden-import openpyxl ^
        --hidden-import plotly ^
        --collect-all streamlit ^
        forobs_launcher.py

    Output: dist/FOROBS.exe  (~150-300 MB)

NOTE: PyInstaller with Streamlit can be tricky. Alternative tools:
  - Nuitka (compiles to native C, harder to reverse-engineer)
  - cx_Freeze (similar to PyInstaller)
  If PyInstaller has issues, try:
    pip install nuitka
    nuitka --standalone --onefile --include-package=streamlit forobs_launcher.py

2.6 Test the build
-------------------
    cd dist
    copy ..\card_settings.json .
    copy ..\card_layout.json .
    FOROBS.exe
    → Browser should open at localhost:8501


3. DEPLOY TO SHIP PC
======================

3.1 Copy to USB stick:
      FOROBS.exe
      card_settings.json
      card_layout.json
      (optionally: logbook.db if migrating data)

3.2 On ship PC: copy to Desktop or any folder

3.3 Double-click FOROBS.exe
    → Black console window appears: "Starting... please wait"
    → Browser opens automatically in ~3 seconds
    → App is running

3.4 To stop: close the console window (X button)


4. UPDATING THE APP
=====================

When app.py changes:
  - Rebuild FOROBS.exe on the build PC (repeat section 2.5)
  - Copy new FOROBS.exe to ship PC via USB
  - logbook.db is NOT inside the exe — data is preserved


5. NETWORK ACCESS (optional)
==============================

If another PC is on the same ship LAN/WiFi, it can access the
running app without its own copy:

  On ship PC running FOROBS.exe, find IP:
    Open cmd → ipconfig → look for IPv4 Address

  On other PC, open browser:
    http://<ship-pc-ip>:8501

  The --server.address 0.0.0.0 flag is already set in the launcher
  to allow network access.


6. PROTECTION SUMMARY
=======================

  ✓ No source code visible — everything compiled into .exe
  ✓ No Python installation exposed — bundled inside binary
  ✓ No editable .py files — prevents "creative modifications"
  ✓ Data files (db, json) are external — easy to backup/migrate
  ✓ No admin rights needed to run
  ✓ No internet needed on ship PC


7. TROUBLESHOOTING
====================

Problem: "Windows protected your PC" SmartScreen warning
  → Click "More info" → "Run anyway" (first run only)

Problem: Browser doesn't open
  → Manually open http://localhost:8501 in any browser

Problem: Port 8501 in use
  → Close any other FOROBS instance, or wait 30 seconds and retry

Problem: App crashes immediately
  → Run from cmd to see error: cmd → cd to folder → FOROBS.exe
  → If db corrupted: delete logbook.db, app creates fresh one

Problem: Antivirus blocks exe
  → Add FOROBS.exe to antivirus exceptions (may need admin help)

Problem: Exe too large (300MB+)
  → Normal for bundled Python apps. Can use --onedir instead of
    --onefile to speed up startup (creates folder instead of single exe)
